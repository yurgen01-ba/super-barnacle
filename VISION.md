# Project Brain Vision

## Vision

Build the best AI Business Analyst in the world.

Not a chatbot. Not a documentation assistant. Not a search engine. Not a meeting summarizer.

A second analyst for the team.

## Mission

Every software project should have a digital brain.

Project Brain should understand:

- how the product works;
- why decisions were made;
- who is responsible for what;
- which requirements exist;
- which processes are affected by changes;
- which documentation is outdated;
- which risks are known;
- which parts of the project are unclear.

## Product thesis

Most AI knowledge tools answer:

```text
What was written?
```

Project Brain should answer:

```text
How does the project work?
```

## Long-term goal

Project Brain becomes a digital twin of the project.

A user should be able to ask:

```text
If we change the Funding process, what requirements, APIs, modules, tests,
Jira tickets, Confluence pages and business rules are affected?
```

And the system should answer using a connected model of the project.

## Principles

- Project first.
- Knowledge first.
- Graph first.
- Reasoning before generation.
- Explainability over fluency.
- Every answer must have provenance.
- Every artifact must be reproducible.
- Knowledge evolves.

## What Project Brain is not

Project Brain is not:

```text
A generic chatbot
A simple RAG application
A transcript summarizer
A Confluence clone
A Jira clone
A note-taking app
```

Project Brain is a reasoning layer over project knowledge.

## Strategic differentiator

Project Brain should not compete by having better chat.

It should compete by having a better internal model of the project.
